A modern JavaScript application focused on building scalable and maintainable code using ES6 modules and modern development tooling.

> **Note:** This project is part of my progression through "The Complete JavaScript Course". After mastering asynchronous JavaScript, this section focuses on structuring real-world applications using modules, bundlers, and development workflows.

---

## ✨ Features

- **Modular Architecture:** Splitting code into reusable, maintainable ES6 modules.
- **Separation of Concerns:** Clear distinction between data logic, UI, and application control flow.
- **NPM Integration:** Managing third-party packages and dependencies.
- **Module Bundling:** Using tools like Parcel to bundle and optimize assets.
- **Hot Module Replacement (HMR):** Live development without full page reloads.
- **Production Build Optimization:** Minification, tree-shaking, and performance improvements.

## 🧠 What I Learned

This project marks a transition from writing scripts to engineering structured applications:

1. **ES6 Modules:** Using `import` and `export` to organize code across multiple files.
2. **Project Architecture:** Structuring applications using patterns (MVC-inspired approach).
3. **NPM Ecosystem:** Installing, managing, and using external packages.
4. **Bundlers (Parcel):** Understanding how modern tools bundle assets and improve performance.
5. **Babel & Transpiling:** Writing modern JS while maintaining browser compatibility.
6. **Development vs Production:** Differentiating environments and optimizing builds.

## 🚀 How to Run Locally

1. Clone this repository:

```bash
git clone https://github.com/fahri238/js-modules-tooling-functional.git
```

2. Install dependencies:

```bash
npm install
```

3. Start development server:

```bash
npm start
```

4. Build for production:

```bash
npm run build
```
